  - 输出 HTML 格式，带 emoji 和柱状图

- `f2b_manager/monitor/health.py` — HealthChecker 类
  - check(): fail2ban-client ping → systemd 状态查询 → 自动重启
  - 重试上限 3 次（MAX_RESTART_ATTEMPTS），超限后仅告警不重试
  - 重启后等待 3 秒稳定 → 再次 ping 验证
  - 每次状态变化发送 Telegram 告警（检查 enable_health_alert 开关）
  - bot=None 不崩溃，仅记录日志

- `f2b_manager/monitor/scheduler.py` — F2BScheduler 类
  - 基于 APScheduler 3.11 AsyncIOScheduler
  - 注册 4 个定时任务:
    1. daily_report (cron, 从 config 读取时间)
    2. weekly_report (cron, 从 config 读取星期+时间)
    3. poll_ban_changes (interval, 5 分钟可配置)
    4. health_check (interval, 10 分钟可配置)
  - 轮询兜底 diff 算法:
    - 遍历 jail → get_jail_status() → 构建 (ip, jail) 集合
    - 与 db.get_current_bans() 快照做 set diff
    - added → 构造 BanEvent(BAN) → alert_sender.send_ban_alert()
    - removed → 构造 BanEvent(UNBAN) → alert_sender.send_ban_alert()
    - 更新 db.set_current_bans() 快照
  - start() + shutdown(wait=True) 优雅生命周期管理
  - 所有时间配置从 config.schedule 读取，支持运行时修改

- `systemd/f2b-manager.service` — Type=simple, User=root, After=network-online.target fail2ban.service, ExecStart=/opt/f2b-manager/venv/bin/python -m f2b_manager, Restart=always/RestartSec=10, 环境变量 PYTHONUNBUFFERED=1 + F2B_CONFIG。

- `docs/deployment.md` — 新手部署文档：系统要求、创建 Bot+获取 chat_id、上传项目、一键安装、填写配置、启动服务、验证、首次配置检查清单、GeoIP 可选配置、升级/重装、路径速查。
