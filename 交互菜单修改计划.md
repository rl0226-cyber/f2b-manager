### Telegram Bot 引导式配置流程（功能 4 的核心）

```
=== Telegram Bot 配置向导 ===

【第 1 步】创建 Bot
  1. 打开 Telegram，搜索 @BotFather
  2. 发送 /newbot
  3. 按提示输入 Bot 名称和用户名
  4. 复制返回的 Bot Token（格式如 123456789:ABCdefGHIjkl...）

  请粘贴 Bot Token:
  > █

  ✓ Token 格式校验通过

【第 2 步】获取你的 User ID
  1. 在 Telegram 搜索 @userinfobot
  2. 给它发任意消息
  3. 它会回复你的 User ID（纯数字，如 123456789）

  请输入你的 User ID:
  > █

  ✓ User ID 已记录

【第 3 步】验证连接
  正在发送测试消息到你的 Telegram...
  ✓ 测试消息发送成功！请打开 Telegram 查看

【第 4 步】保存配置
  配置已保存到 /etc/f2b-manager/config.yaml
  服务将在 3 秒后自动重启以应用新配置...
```

### `f2b` 快捷指令

安装时创建符号链接：
```bash
ln -sf /opt/f2b-manager/venv/bin/f2b-manager /usr/local/bin/f2b
```
用户在终端任意位置输入 `f2b` 即可启动管理菜单。

### install.sh 修改要点

当前 install.sh 末尾是打印"下一步操作"文字提示用户手动编辑配置。改为：
1. 删除末尾的"下一步操作"文字提示
2. 添加 `f2b` 快捷指令创建（symlink）
3. 安装完成后自动执行 `f2b-manager menu`（交互式引导配置）
4. 配置完成后提示启动服务

## 三、提示词

> 在新窗口选择 **DeepSeek-V4-Pro** 模型，复制以下提示词粘贴执行：

```
你是一个 Python 后端开发专家。请帮我为 f2b-manager 项目添加交互式管理菜单功能。这个项目是一个 VPS Fail2ban 管理系统，目标用户是 VPS 新手，需要通过交互式菜单（而非手动编辑配置文件）完成所有操作。

## 项目背景
项目已完成全部代码开发（M0-M6），包含 fail2ban 管理、Telegram Bot、实时预警、定时监控等模块。你现在需要新增一个交互式菜单模块，让新手用户通过数字选择菜单完成所有操作。

## 必读文件（请先全部读取再开始编码）
1. f2b_manager/cli.py — 当前 CLI 入口，有 run/fail2ban/notify/status 子命令，你需要添加 menu 子命令
2. f2b_manager/config.py — 配置加载模块，目前只有 load_config() 读取，你需要新增 save_config() 写入功能
3. f2b_manager/app.py — 应用主类，了解各模块如何组装
4. f2b_manager/fail2ban/manager.py — Fail2banManager，菜单中调用其 get_status/get_banned_ips/ban_ip/unban_ip
5. f2b_manager/fail2ban/installer.py — Fail2banInstaller，菜单中调用 install/uninstall/update
6. f2b_manager/storage/models.py — 数据模型，了解 Fail2banStatus/JailStatus 等返回类型
7. install.sh — 当前安装脚本，末尾有"下一步操作"文字提示，你需要改为自动调用菜单
8. config/config.example.yaml — 配置模板，了解 YAML 结构

## 需要完成的任务

### 任务 1：新增 f2b_manager/menu.py（核心）
创建交互式菜单模块，包含以下功能：

主菜单（数字选择，循环显示直到退出）：
  1. 安装 Fail2ban — 调用 Fail2banInstaller.install()，显示进度和结果
  2. 卸载 Fail2ban — 调用 Fail2banInstaller.uninstall()，二次确认
  3. 更新 Fail2ban — 调用 Fail2banInstaller.update()，显示版本变化
  4. 配置 Telegram Bot 通知（引导式）— 这是最重要的功能：
     - 分步引导用户创建 Bot（显示图文教程）
     - 输入 Bot Token（校验格式：数字:字母数字字符串）
     - 输入 User ID（校验为纯数字）
     - 可选：输入额外操作员 chat_id
     - 调用 Telegram API 发送测试消息验证 Token 有效
     - 验证通过后调用 save_config() 写入配置文件
     - 提示是否立即重启服务
  5. 查看运行状态 — 调用 Fail2banManager.get_status()，格式化显示
  6. 查看封禁 IP 列表 — 调用 get_banned_ips()，表格显示
  7. 手动封禁/解封 IP — 子菜单：封禁（输入IP+jail）/解封（输入IP）
  8. 启动/停止/重启服务 — 子菜单，调用 systemctl 命令
  9. 查看日志 — 调用 journalctl -u f2b-manager --no-pager -n 50
  0. 退出

技术要求：
- 使用 ANSI 颜色码美化输出（绿色成功、红色错误、黄色警告、青色标题）
- 中文界面，对新手友好，每个操作都有清晰提示
- 输入校验：Token 格式、IP 格式、数字范围
- 错误处理：操作失败时显示友好错误信息，不崩溃
- 菜单类名为 InteractiveMenu，构造函数接收 config_path 参数
- run() 方法为主循环
- 各功能为独立方法 _menu_install() / _menu_config_telegram() 等
- 调用 fail2ban 相关操作时用 try/except 处理模块未就绪的情况
- 配置 Telegram Bot 时，测试消息通过 httpx 直接调用 Telegram API（GET https://api.telegram.org/bot<token>/sendMessage?chat_id=<id>&text=测试消息），不依赖 Bot 模块

### 任务 4：修改 install.sh

2. 添加 f2b 快捷指令创建：
   ```bash

   ln -sf "$VENV_DIR/bin/f2b-manager" /usr/local/bin/f2b

   ```
3. 安装完成后自动启动交互式菜单：
   ```bash
