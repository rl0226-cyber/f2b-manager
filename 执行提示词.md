**推荐模型：DeepSeek-V4-Pro**（积分系数 0.16，性价比最高）

**提示词：**

```
你是一个 Python 后端开发专家。请帮我完成 f2b-manager 项目的 Fail2ban 管理模块（M1）。

## 项目背景
这是一个 VPS Fail2ban 管理系统，集成 fail2ban 安装/卸载/更新 + Telegram 机器人。项目已完成 Wave 1（M0 基础设施层），你现在需要基于已有的接口契约实现 Fail2ban 管理模块。

## 必读文件（请先全部读取再开始编码）
1. 设计方案.md — 完整设计文档，重点看「4.1 Fail2ban 管理模块」和「4.4 实时预警模块」中的 telegram-notify.conf 部分
2. f2b_manager/storage/models.py — 接口契约定义，你必须实现其中的 IFail2banManager 和 IFail2banInstaller 协议

1. f2b_manager/fail2ban/parser.py — 解析 fail2ban-client 命令输出
2. f2b_manager/fail2ban/manager.py — 实现 IFail2banManager 接口（运行时管理）
3. f2b_manager/fail2ban/installer.py — 实现 IFail2banInstaller 接口（安装/卸载/更新）
4. f2b_manager/fail2ban/config_builder.py — 生成 jail.local 配置文件
5. config/jail.local.template — jail 配置模板（含 telegram-notify action 引用）
6. config/telegram-notify.conf — fail2ban 自定义 action 配置

## 技术要求
- parser.py: 解析 `fail2ban-client status` 和 `fail2ban-client status <jail>` 的输出，需兼容 fail2ban 0.11.x 和 1.0.x 两种输出格式。返回 models.py 中定义的 Fail2banStatus、JailInfo、JailStatus 数据类。
- manager.py: 实现 IFail2banManager 的全部方法（get_status/get_jails/get_jail_status/get_banned_ips/ban_ip/unban_ip/reload），通过 run_command() 调用 fail2ban-client。
- installer.py: 实现 IFail2banInstaller 的全部方法（install/uninstall/update）。安装流程：检测发行版→用对应包管理器安装→生成 jail.local→部署 telegram-notify action→部署 notify.sh→systemctl enable。卸载时保留配置备份。更新时记录版本变化。
- config_builder.py: 基于 Fail2banConfig 生成 jail.local 内容，关键是在 action 中追加 telegram-notify action。
- 所有方法要有完善的错误处理和日志记录（用 from ..utils.logger import get_logger）。

## 验收标准
- 代码能通过 `python -c "from f2b_manager.fail2ban.manager import Fail2banManager"` 导入
- parser.py 能正确解析示例的 fail2ban-client 输出（请在代码中附带测试用的示例输出字符串）
- 在没有 fail2ban 的环境下（如 macOS），代码不会在导入时崩溃，只在调用时报错
- 严格遵循 models.py 中的接口签名，不要随意更改返回类型

## 项目背景

- formatters.py: 封装状态、封禁列表、报告等的 HTML 格式化

## 重要：Mock 依赖

```

## 项目背景
这是一个 VPS Fail2ban 管理系统。当 fail2ban 封禁 IP 时，通过自定义 action 调用 notify.sh 脚本，脚本再调用 f2b-manager CLI 的 notify 子命令，将事件转发给守护进程，由预警模块构造消息并发送到 Telegram。项目已完成 Wave 1（M0 基础设施层）。

## 必读文件（请先全部读取再开始编码）

2. f2b_manager/storage/models.py — 接口契约，你需要实现 IAlertSender 协议，依赖 IMessageSender 和 IStateDB
3. f2b_manager/config.py — 了解 NotifyConfig（enable_ban_alert/geoip_enabled/dedup_window_seconds 等）
4. f2b_manager/cli.py — 看 notify 子命令的处理逻辑，了解事件如何传入

1. f2b_manager/notify/geoip.py — IP 归属地查询（本地 mmdb + 在线 API 回退）
2. f2b_manager/notify/dedup.py — 消息去重限流（同 IP 时间窗口内只通知一次）
3. f2b_manager/notify/sender.py — 实现 IAlertSender 协议（构造消息 + 调用 bot 发送）
4. scripts/f2b-notify.sh — fail2ban action 调用的桥接脚本（bash）

- sender.py 的 AlertSender 类实现 IAlertSender 协议（send_ban_alert/send_service_alert）
- 构造函数接收 config、bot（IMessageSender，可为 None）、db（StateDB）参数
- send_ban_alert: 收到 BanEvent 后 → 查询 IP 归属地 → 去重检查 → 构造 HTML 消息 → 调用 bot.send_alert → 记录到 db
- 消息模板参考设计方案 4.4.3 节的 BAN_TEMPLATE 和 UNBAN_TEMPLATE，包含 jail/ip/归属/失败次数/时间/匹配日志

- notify.sh: bash 脚本，被 fail2ban action 调用，参数为事件类型/IP/jail/failures/matches，通过调用 `f2b-manager notify` 子命令转发事件。脚本必须永远 exit 0（避免影响 fail2ban 正常工作）。
- telegram-notify.conf: fail2ban action 配置，定义 actionban/actionunban/actionstart/actionstop 调用 notify.sh
- 所有异步方法正确使用 async/await

## 验收标准
- 代码能通过 `python -c "from f2b_manager.notify.sender import AlertSender"` 导入
- AlertSender 严格实现 IAlertSender 协议签名
- 去重逻辑正确：同 IP 5 分钟内多次 ban 只发送一次通知
- GeoIP 双模式回退正确
- notify.sh 和 telegram-notify.conf 格式正确，可直接部署到 /etc/fail2ban/
- bot 为 None 时不崩溃，只记录日志（便于独立开发）

## 技术要求

  - weekly_report(): 生成每周报告，含 7 天趋势
  - instant_report(): 即时报告，供 /report 命令调用
  - 数据来源：调用 db.count_bans_today()、db.top_banned_ips()、db.top_banned_countries()、f2b_manager.get_status() 等
  - 构造函数接收 config、f2b_manager（IFail2banManager）、db（StateDB）参数

- health.py 的 HealthChecker 类:
  - check(): 执行 fail2ban-client ping，检查 systemd 服务状态
  - 若 fail2ban 异常 → 尝试 systemctl restart fail2ban → 通知（调用 bot.send_alert）→ 记录日志
  - 重试上限 3 次，超过则只通知不重试
  - 构造函数接收 config、bot（IMessageSender）参数

- scheduler.py 的 F2BScheduler 类:
  - 使用 APScheduler 的 AsyncIOScheduler
  - 注册 4 个任务：每日报告（cron）、每周报告（cron）、轮询兜底（interval 5分钟）、健康检查（interval 10分钟）
  - 轮询兜底逻辑：调用 f2b_manager.get_banned_ips() → 与 db.get_current_bans() 对比 → 新增的 IP 调用 alert_sender.send_ban_alert() 补发通知 → 更新 db.set_current_bans()
  - 构造函数接收 config、f2b_manager、bot、alert_sender、db 参数
  - shutdown() 方法优雅关闭调度器
  - 关键：AsyncIOScheduler 必须与 python-telegram-bot 的 Application 共享同一事件循环

## 验收标准

- 健康检查的自动重启逻辑有重试上限
